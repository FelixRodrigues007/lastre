# Lastro Composition Rules

## Layout principles

1. **Use proof trails, not decorative chains.** Lines should communicate sequence: origin → seal → Casper → verdict.
2. **Keep one dominant message.** If an ad needs two headlines, split it into two variants.
3. **Show evidence texture.** Use coordinates, hashes, verdict chips, and contract language as secondary detail.
4. **Balance dark space with precise highlights.** The brand should feel like infrastructure, not nightlife crypto.
5. **Use Invalid intentionally.** Red is not “failure”; it is recorded negative proof.

## Standard composition

```text
[Eyebrow badge]

Large headline
Support sentence

[Proof rail / verdict pair / hash chip]

Lastro mark           RWA provenance trust layer
```

## Proof rail pattern

Use three or four steps:

```text
Origin artifact → SHA-256 seal → Casper attestation → Valid/Invalid verdict
```

Recommended colors:
- Origin: Paper / neutral.
- Seal: Seal gold.
- Casper/network: Network blue.
- Valid: Provenance green.
- Invalid: Signal red.

## Type scale for 1200×627

| Element | Size | Weight | Notes |
|---|---:|---:|---|
| Eyebrow | 18–22px | 600 | Uppercase or small caps |
| Headline | 64–92px | 700 | Tight line-height, 2 lines max |
| Support | 24–30px | 400/500 | 1 sentence |
| Hash/metadata | 16–18px | 500 | Monospace |
| Footer | 18–20px | 500 | Mark + category |

## Do / don't

### Do
- Use: “Proof before token.”
- Use: “Invalid provenance is recorded too.”
- Use: “LLM decides action, not verdict.”
- Use fictional samples only.
- Use concise, technical phrasing.

### Don’t
- Don’t use yield/ROI/returns language.
- Don’t use real mining companies or real people.
- Don’t make the LLM visually look like a judge of truth.
- Don’t imply production readiness unless verified.
- Don’t overcrowd small formats with full architecture diagrams.
