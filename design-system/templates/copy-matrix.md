# Lastro Advertising Copy Matrix

Use these angles to create ad variants without drifting into hype or investment language.

## Approved claims

- “Proof before token.”
- “The chain of proof from land to token.”
- “Verified offline. Anchored on Casper.”
- “The seal decides the verdict. The LLM decides the action.”
- “Invalid provenance is recorded too.”
- “A trust layer for RWA provenance.”

## Banned claims

- “Earn yield from real assets.”
- “Unlock ROI from tokenized minerals.”
- “Guaranteed provenance.”
- “AI verifies truth.”
- “Invest in tokenized mines.”
- Any real company/person/mine names.

## Core angles

### 1. Proof before token
**Audience:** RWA builders, protocol evaluators.

Headlines:
- Proof before token.
- Tokenization needs origin proof.
- Don’t tokenize before proof.
- The proof layer for RWA origin.

Support lines:
- Lastro anchors deterministic provenance attestations on Casper.
- A token is only as credible as the origin proof behind it.
- Before tokenization, Lastro checks the chain of proof.

### 2. Rejection as evidence
**Audience:** Compliance/risk, technical evaluators.

Headlines:
- Invalid is proof too.
- Rejections should be recorded.
- The failed proof path matters.
- Don’t discard bad provenance.

Support lines:
- Lastro records Valid and Invalid outcomes on-chain.
- A rejected lot becomes auditable evidence, not a hidden error.
- Provenance workflows need the negative case, not only the demo path.

### 3. LLM is not truth
**Audience:** AI skeptics, technical buyers.

Headlines:
- The LLM does not decide truth.
- AI chooses action. The seal decides verdict.
- Keep AI out of the verdict.
- Deterministic proof beats AI rubber-stamps.

Support lines:
- Lastro separates agentic decisions from deterministic provenance checks.
- The workflow can use an LLM without making it the source of truth.
- The SHA-256 seal decides Valid or Invalid; the agent decides what to do next.

### 4. Casper-native anchoring
**Audience:** Casper ecosystem, hackathon judges.

Headlines:
- Provenance anchored on Casper.
- Casper-native proof of origin.
- Offline proof. On-chain attestation.
- From seal to Casper verdict.

Support lines:
- Lastro uses Odra/Rust contracts to store proof-of-origin attestations.
- The offline seal is anchored through a Casper Testnet workflow.
- A real contract records the provenance verdict.

## CTAs

Use low-hype technical CTAs:
- Review the architecture.
- Explore the demo.
- See the proof flow.
- Read the contract.
- Inspect the testnet proof.

Avoid conversion CTAs implying financial participation:
- Invest now.
- Get returns.
- Buy the token.
- Join the sale.
